Anatomy of a URL (GET Request):

http:// localhost: 8080/ Jee7Tutorials/ index.html ?id=5&v=7

Let's Get Started
Key Concepts

When you type in a web address (URL) into your browser an hit ENTER, you are sending a request to a web server for information in the form of a web page or server-side program. In addition to what is being requested, you can also send data to to server in the form of QueryString Parameters or form input.

Web clients (browser) communicate with with web servers in a series of request-response cycles. In the request phase, if requesting a regular web page (".html") the server will look for the page and return a 404 error if not found, or a stream of bytes representing the code in the page file (but the file iteself). This is called the "Response" phase. Finally, the browser interprets this code and renders the web page as a result. The fact that at least two computers are involved in this (client and server) is why the web is inherently a distributed computer environment.

If the page name is suffixed by something other than ".html" -- for example ".jsp" meaning the page contains some java code -- then the server will look for the java and execute it first, and then modify the html in the byte stream to contain any results from the execution of that code. Works the same way for PHP and ASP.NET and other web technologies.

  1. This is the transport protocol, which determines the rules for communication between client and server. Others are ftp:// and telnet:// e.g.

  2. This is the server address, which is usually based on a domain name. The 'www' part is optional and must be configured by a server administrator. And it doesn't have to be 'www' alone. Try this one: http://sandbox.byteshop.net 'localhost' is a built-in name for a developer's own machine. This could also be an IP address like 127.0.0.1

  3. This is the port number for a developer web server. Production servers usually use the default value of 80, and then it isn't needed in the URL. Port numbers allow for multiple services to be running on a single machine -- even multiple web servers.

  4. This is the web application name, or folder name within a web app. Folders can be nested to any depth. If the web app is installed at the root, there is no web app name, just the domain name. By using web app folders you can have many web apps on a single server.

  5. This is the name of the web page being requested. Or, it could be the name of a program running on the server. File name extensions are not limited to '.html' Others include '.jsp', '.php', '.aspx', '.pl', '.cgi' and many more. The extension determines what the server does upon receiving the request.

  6. This is optional data that can be passed to the server with the request. It's called a QueryString, begins with a '?' symbol and has key=value pairs called 'parameters.' Multiple pairs are separated by the ampersand symbol. Programs running on the server can read these and use them in data processing.
