How the Server Responds to GET Requests
Containing QueryString Parameters

Given this HTTP GET request:

http:// localhost:8080/ Jee7Tutorials/ dynamicPage.jsp ?name=Butkus

Just as in the previous demo the client (browser) sends an HTTP GET request to the server for a page file name. The server then looks for the file in the folder Jee7Tutorials. But this time something very different happens when the server finds the file. Because it ends with the extension ".jsp" the server looks for a matching processing engine -- in this case a JavaServer Page engine (JSP). Finding that it parses (looks through) all the source code in the file, looking for Java code mixed in with the HTML. When it finds this code it immediately executes it and adds any resulting text to the HTML, effectively modifying the original page. Then and only then does the server send the response back to the client. The response is different now, click the file name above to see the response.

The code highlighted in green is java code mixed in with the HTML. This is the code that gets executed immediately on the server after it responds to the request for the page.

It's job is to take the QueryString parameter value, which gets sent along with the request (name = "Butkus") and use that value in the HTML response, effectively modifying the HTML code in process.
